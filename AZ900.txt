-------------------------------------------------------------------------------------------------------------------------------

雲端：透過網際網路傳遞的運算服務

        雲端服務包含：伺服器、儲存體、資料庫、網路、軟體、分析和情報

 

        計算能力 Compute Power -

                Linux Server, Web app

        存儲 Storage -

                File and Databases

        網絡 Networking -

                VPN, Peering

        分析 Analytics -

                Visualize Telemetry & Performance Data

資本支出 Capital Expenditure (CapEx)-

        Upfront expenditure on physical infrastructure

        物理基礎設施的前期支出

        預先在實體基礎結構上花費，經過一段時間再扣除這筆預先費用。

 

營業費用 Operation Expenditure (OpEx)-

        No upfront cost pay as you go model

        無需預付費用，即用即付模式

       

        All cloud providers are OpEx model

        所有云提供商都是運營支出模式

        是服務或產品的花費，且目前正在支付這筆費用。可在花費的同一年扣除此費用。

 

----------------------------------------------------

公有雲 Public Cloud -

        No Capital expenditure 無資本支出

        Increased Agility  提高敏捷性

        Pay as you go model / OpEx model 付費模式/營運支出模式

        No maintenance required  無需維護

        Fewer in-house skill needed. 所需的內部技能更少

       

        Less Control over security 對安全性的控制較少

        Unmet compliance requirement 未滿足合規性要求

        Lack of ownership can be seen as disadvantage 缺乏所有權可被視為不利因素

 

私有雲 Private Cloud -

        Control - Retained by org. 控制 - 由組織保留。

        Security - Controlled by org. 安全性 - 由組織控制。

        Compliance - managed by org. 合規性 - 由組織管理。

 

        Upfront CapEx cost 前期資本支出

        Less Agile 更少敏捷

        Maintenance and skillset needed. 維護和技能組合

 

混合雲 Hybrid Cloud -

        Flexible solution 靈活的解決方案

        Cost Management 成本管理

        Security & Control 安全與控制

        Compliance 合規性

 

        Upfront CapEx 前期資本支出

        Maintenance and Skillset needed 維護和所需技能

        Management Complexity 管理複雜性

       

        雲端類型：

                公用雲端：無資本支出，屬於營運支出。服務透過公用網際網路提供，任何人都可以使用。

                              雲端資源由協力廠商雲端服務提供者所擁有及操作，並透過網際網路傳遞

                私人雲端：為資本支出，同時有營運支出(維修費)。由某個企業或組織的使用者以獨佔方式使用的運算資源所組成。

                          私人雲端可實際位於組織本地(內部部署)資料中心或透過協力廠商服務提供者裝載。

                混合式雲端：是一種運算環境，可讓資料和應用程式在公用雲端與私人雲端之間共用，藉以結合這二種雲端。

 

 

        雲端模型比較：

                公用雲端：沒有會擴大的資本支出、可以快速佈建及取消佈建應用程式、組職只需為其使用的項目支付費用。

                私人雲端：一開始必需購買硬體，而且之後需要維護。組織可以完全掌控資源與安全性。組織負責硬體維護及更新。

                混合式雲端：提供最大的彈性、組織決定要執行其應用程式的位置。組職控制安全性、合規性或法律需求。

 

       

        關鍵考題：減少資本支出：選公有雲

         最大彈性：混合雲

         公與私的網路能一起：混合雲  (意思是講私的不能連外、公的不能連私的都是錯的答案)

         公有雲比較不安全：錯 (一般反而更安全)

         公有雲的資源/網路，人人都可以存取：錯 (依自己的設定可開放或不開放或限定…)

                雲端運算是以使用"量"為基礎的模型

Share Responsibility Model (雲端服務模型 )-

 

IaaS - 基礎結構即服務。最接近管理實體服務器。雲端提供者將硬體保持在最新狀態，作業系統及網路設定由雲端租用戶管理。

        成本 Costs :

                No upfront  無預付款

                Pay as you go 按量付費

        用戶所有權 User Ownership :

                Customers are responsbile for purchase installation configuration and management of their own OS app & Middleware

                客戶負責購買、安裝、配寘和管理自己的作業系統應用程序和中介軟體

        雲提供商所有權 Cloud Provider Ownership :

                Cloud provider is responsible for underline infrastructure 雲提供商負責強調基礎設施

        常見用例 Common Use Case :

                Workload Migration , Dev & Testing , Website Hosting, Backup & Recovery

                工作負載遷移、開發和測試、網站託管、備份和恢復

                Azure Storage、Azure VPN、Azure Network、VM (可以理解為就是租硬體開始起算)

               

PaaS - 平台即服務。雲端提供者負責管理虛擬機器與網路資源，租用戶負責應用程式佈署到受控的主控環境。

        Provide an env that can be used to quickly build test and deploy app. 提供可用於快速構建測試和部署應用程序的環境。

        No need to concern abt infrastructure 無需擔心abt基礎設施

       

        Costs :

                No upfront

                Pay as you go

               

        User Ownership :

                Customers are responsbile for their own app development.

                客戶對自己的應用程序開發負責。

               

                Customers are not responsbile for managing infrastructure or Servers

                客戶不負責管理基礎架構或服務器

               

        Cloud Provider Ownership :

                Cloud provider is responsible for OS management network service configuration

                雲提供商負責作業系統管理網路服務配寘

               

              Cloud provider is responsbile for everything except application that customer want to run.

                雲提供商負責除客戶希望運行的應用程序之外的一切

               

        Common Use Case :

                Development Framework, Analytics and BI,

                開發框架、分析和BI

                Azure SQL Database、Azure App Service、Azure Function、DNS、Azure Backup、Azure Iot Hub

 

SaaS - 軟體即服務。雲端提供者會管理應應用程式環境有關的所有事項。租用者只需要向雲端提供資料。

       例如Office 365可在雲端完整執行，使用者只需要建立內容。

          

        SaaS provides centrally hosted and managed software for end users.Who access software over internet

        用戶提供集中託管和管理的軟件。 誰通過互聯網訪問軟件

       

        Eg. MS-Office 365, Emails., MS-Team, MS dynamic CRM online

            Office365、Outlook、Azure Iot Central、文字轉語音，使用者只需要輸入資料。

 

        Costs :

                No upfront

                Subscription based 基於訂閱

               

        User Ownership :

                Customers only uses app

                客戶僅使用應用程序

               

                Customers are not responsbile for managing and maintenance of app

                客戶不負責管理和維護應用程序

               

        Cloud Provider Ownership :

                Cloud provider provides provision management of app

                雲提供商提供應用程序的供應管理

               

        Common Use Case :

                Development Framework, Analytics and BI, 開發框架、分析和BI

Key Concepts -

        高可用性 High Availability(HA)- Ability to keep busniess up and running without or minimum downtime

                                     能夠在沒有或最小化停機時間的情況下保持業務正常運行

        可擴展性 Scalability - To increase or decrease resources to workload

                               增加或減少資源工作負載 (垂直方式：新增虛擬機器的RAM或CPU; 水平方式：新增資源的執行個體(如vm))

        彈性 Elasticity - Auomatic way of increase or decrease resources to workload

                          增加或減少工作負載資源的自動方式(可以設定自動縮放，讓應用程式一律擁有所需要的資源)

        靈活度 Agility - Ability to react quickly

                         快速反應的能力(應用程式需要變更時，快速部署及設定雲端式資源。)

        容錯 Fault Tolerance - Remain running despite failure (Built in Redundancy)

                               儘管出現故障仍保持運行（內置冗餘）

        災難復原 Disaster Recovery - Recover from an events that take down a cloud service

                                     從導致雲服務中斷的事件中恢復(利用雲端式備份服務、資料複寫與地理位置散發來避免災害時的損失。)

        Global Reach - Reach users all over the world

                       可部署到世界個地的區域資料中心，藉此確保在其區域中一律擁有最佳效能。

        預測成本 Predictive Cost -

        技術技能 Tech Skill - Allow customer to quickly deploy solutions without in-house hardware expertise

                              允許客戶在沒有內部硬件專業知識的情況下快速部署解決方案

        提高生產力 Increased Productivity - No need to racking and staking of hardware

                                            無需擱置和抵押硬件

        Security -

 

規模經濟 Economies of Scale -

        Ability to reduce costs while improving efficiency as a result of operating at a large scale.

        能夠通過大規模運營降低成本同時提高效率。

       

基於消費的模式 Consumption based model -

        Incur no up-front cost

        不產生預付費用

       

        Costly infrastructure not needed.

        不需要昂貴的基礎設施

       

        Pay for additional resources as needed.

        根據需要支付額外資源

       

        Stop paying for resources that no longer needed.

        停止為不再需要的資源付費

                原則上虛擬環境。或者像輕量化的虛擬機器。

Azure容器執行個體：最簡單快速，無需管理任何虛擬機器，即可在Azure中執行容器。此為PaaS供應項目。

                           簡單講就是一個現成的平台，上面直接執行你的服務，其他你都不管。

                                       

Azure Kubernetes：對大量容器進行自動化、管理及互動的工作稱為「協調式流程」。適用具有分散式架構的容器。

                            基本上就是一個更高階的管理系統，如同vmware的vCenter一樣，可以對容器作設定、管理，包含轉移、複製等等容錯機制。

                    vCneter管的是VM(想成完整的OS)，Kubernetes管理的是容器(想成命令列模式的OS)

         

Azure Virtual Desktop：在雲端上執行的桌面和應用程式虛擬化。

                可執行的動作：

                        設多多會話Windows 11或Windows 10

                        呈現Microsoft 365 Apps企業板，並將其優化，以在多使用者虛擬案例中執行

                        具有免費擴充安全性更新的Windows 7虛擬桌面

                        將現有的Remote Desktop Service (RDS)和Windows Server Desktop和應用程式帶到任何電腦

                       

                連線數授權與你自己已有的有關，如果沒有現成的，線上是by人計價。

                                使用方式有二種：個人和集中式(Pool共享)。連線數和資源都是自由設定的，沒有限制。

                                個人是指每個人都有一個獨立的資源。Pool是比如開100個給公司150個人使用。

                                (不會同時在線的工作方式就適用，比如四班二輪，A/B班不會同時使用，連線數就不需要用到1比1。)

Azure Network Service -

 

        Azure Virtual Network  Azure 虛擬網絡, 可讓Azure資源從此通訊，或與網際網路上的使用者及內部部署用戶端電腦通訊。

    提供下列重要網路功能：

                        隔離和分割：可以創建多個隔離的虛擬網路。公共IP範圍僅存在於虛擬網路中，不可通過Internet路由。(可管控多個Region/Vnet, 過濾網址)

                        網際網路通訊：

                     Azure資源之間的通訊：可以是資源如VM、App Service、Kubernetes Service，也可以是服務端點，如Azure SQL資料庫或是儲存體

                        與內部部署資源通訊：三種機制 點對站虛擬私人網路(P2S)、站對站虛擬私人網路(S2S)、Azure ExpressRoute

                        路由網路流量

                        篩選網路流量：使用網路安全性群組(使用設定規則的方式)和網路虛擬設備(使用如防火牆等的虛擬設備)的方法來篩選子網路之間的流量。

                        連線虛擬網路：將虛擬網路連結在一起   

                       

                Azure Virtual Network allows several type of Azure resources like Azure VMs to securely communicate with one another with internet

                      and with on prim networks

                Azure 虛擬網絡允許多種類型的 Azure 資源（如 Azure VM）與 Internet和主要網絡安全地相互通信

               

                Scoped to Single region

                範圍為單個區域

               

                Connect multiple Virtual Networks from different regions using virtual network peering

                使用虛擬網絡對等連接來自不同區域的多個虛擬網絡
